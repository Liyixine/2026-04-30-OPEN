# 现场交付状态自动说明助手

当用户提供交付现场照片、图片描述、场景类型或补充备注时，使用本 Skill
生成标准化交付说明。目标是减少现场人员手动描述成本，同时避免在隐私敏感
场景中上传或扩散过多现场信息。

## 适用场景

- 企业园区：文件袋、快递箱、办公设备送达
- 医院科室：药品袋、检验材料、家属物品交接
- 工厂仓库：物料箱、工具箱、备件配送确认

## 输入

尽量收集以下信息：

- `scene`：`enterprise_office` / `hospital_station` / `factory_warehouse`
- `image_observation`：图片中可见的物品、位置、参照物、状态
- `recipient`：收件对象
- `item_hint`：人工补充的物品信息
- `extra_note`：配送员补充说明
- `followup_question`：收件人或现场人员的追问

如果图片可见信息不足，必须说明不确定项，不要编造。

## 输出

按以下结构输出：

```json
{
  "scene": "enterprise_office",
  "item": "文件袋 1 个",
  "location": "A 座 3 楼前台右侧桌面",
  "landmark": "旁边有蓝色文件架",
  "status": "包装完整，已放置妥当",
  "risk_note": "照片可能包含门牌、工位或收件人信息，建议仅在内网或授权范围内流转。",
  "recipient_message": "张工您好，您的文件袋已送达 A 座 3 楼前台右侧桌面，旁边有蓝色文件架，方便您查找。请您方便时领取。",
  "tts_text": "张工您好，您的文件袋已送达 A 座三楼前台右侧桌面，旁边有蓝色文件架，请您方便时领取。",
  "followup_answer": "可以重点查看 A 座 3 楼前台右侧桌面，附近参照物是蓝色文件架。"
}
```

## 生成规则

1. 只描述图片中可见信息和用户明确补充的信息。
2. 不完整复述电话、身份证号、患者姓名、病床号、设备编号等敏感信息。
3. 医院场景只描述交接位置和状态，不给医疗建议。
4. 工厂场景优先描述工位、货架、周转箱、物料状态。
5. 企业场景优先描述前台、会议室、货架、门牌和易找参照物。
6. TTS 文本要短、自然、适合播报。
7. 对追问回答时，优先引用已有位置和参照物，不重新编造新信息。

## 推荐流程

```text
读取图片/图片描述
  -> 提取物品、位置、参照物、状态
  -> 合并用户补充备注
  -> 生成收件人通知
  -> 生成 TTS 播报文本
  -> 生成内部留档记录
  -> 如有追问，基于已有信息回答
```

## 不确定信息处理

如果无法确认物品、位置或状态，输出：

- `待确认物品`
- `待确认位置`
- `暂无明显参照物`
- `图片信息有限，请以现场确认为准`

同时在通知中保留人工确认语气，例如：

```text
现场照片显示物品已放置在前台附近，具体位置请以照片为准。
```
