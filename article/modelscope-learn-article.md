## 二、应用场景说明

在企业园区、医院科室、工厂仓库等环境中，经常存在现场交付任务：

- 文件、快递、办公设备送达到前台或会议室
- 药品、检验材料、家属物品送达到护士站或交接台
- 物料、工具、备件送达到工位、货架或维修区

这些场景有几个共同特点：

1. **弱网或内网环境多**：工厂、医院、政企单位可能无法稳定访问外网。
2. **隐私和安全要求高**：照片可能包含门牌、科室、工位、患者信息、产线设备。
3. **人工描述成本高**：现场人员需要反复说明“东西放在哪里”。
4. **沟通误差常见**：收件人可能找不到物品，需要再次追问。

因此，本项目希望将“拍照说明送达位置”这件事自动化。

核心目标：

```text
交付现场照片
  -> 本地 VLM 理解物品、位置、参照物
  -> 自动生成送达通知
  -> 输出 TTS 播报文本
  -> 支持二次追问
  -> 生成 Markdown 留档
```

## 三、整体方案设计

系统由四层组成：

```text
[交付现场图片]
        ↓
[OpenVINO VLM 图片理解]
        ↓
[交付信息结构化与场景模板]
        ↓
[收件人通知 / TTS 文本 / Markdown 留档 / Skill 输出]
```

### 3.1 场景设计

第一版覆盖三个典型场景：

- 企业园区：文件袋、快递箱、办公设备
- 医院科室：药品袋、检验材料、家属物品
- 工厂仓库：物料箱、工具箱、备件

这些场景比单一外卖场景更适合本地化部署：

- 不依赖平台 API
- 更容易体现内网部署价值
- 隐私和安全理由更充分
- 可用一套通用交付凭证逻辑覆盖多个行业

### 3.2 输出结构

系统最终输出统一结构：

```json
{
  "scene": "enterprise_office",
  "item": "文件袋 1 个",
  "location": "A 座 3 楼前台右侧桌面",
  "landmark": "旁边有蓝色文件架",
  "status": "包装完整，已放置妥当",
  "recipient_message": "张工您好，您的文件袋已送达 A 座 3 楼前台右侧桌面，旁边有蓝色文件架，方便您查找。请您方便时领取。",
  "tts_text": "张工您好，您的文件袋已送达 A 座三楼前台右侧桌面，旁边有蓝色文件架，请您方便时领取。",
  "followup_answer": "可以重点查看 A 座 3 楼前台右侧桌面，附近参照物是蓝色文件架。"
}
```

## 五、从 Baseline 到交付助手的创新改造

官方 Baseline 主要验证单个模型能力：

- VLM 能理解图片
- ASR 能识别语音
- TTS 能合成语音

本项目在此基础上做了应用化改造：

1. **场景收束**
   - 从通用图片问答变成现场交付状态说明。

2. **结构化输出**
   - 不只输出一段描述，而是拆成物品、位置、参照物、状态、风险提示。

3. **通知生成**
   - 根据企业、医院、工厂不同场景生成不同语气的送达说明。

4. **TTS 播报文本**
   - 生成适合语音播报的短文本，方便现场人员快速确认。

5. **二次追问预留**
   - 如果收件人仍然找不到，可继续追问“旁边有什么标志”。

6. **Skill 封装**
   - 将能力封装为 OpenClaw Skill，便于在 Copaw 创空间中复用。

## 八、总结与展望
